## 1️⃣ 上传新音频按钮

### 在哪里找到？
右侧历史记录面板，"📚 历史记录"标题下方，有一个蓝紫色渐变按钮：

```
┌────────────────────────┐
│  📚 历史记录            │
├────────────────────────┤
│  📤 上传新音频         │  ← 点这里！
├────────────────────────┤
│  📄 audio1.mp3         │
│  📄 audio2.mp3         │
└────────────────────────┘
```

### 什么时候用？
- ✅ 想上传新音频文件时
- ✅ 想清空当前显示内容时
- ✅ 查看完历史记录，准备处理新文件时

### 效果
点击后页面立即：
- 清空转录和总结区域
- 清空所有对话记录
- 回到默认上传状态
- 平滑滚动到顶部

**再也不用刷新页面了！** 🎉

## 2️⃣ 删除历史记录

### 如何删除？

**步骤：**
1. 将鼠标移到任意历史记录上
2. 右上角出现红色 **×** 按钮
3. 点击 × 按钮
4. 弹出确认框
5. 点击"确定"删除

```
悬停前：
┌────────────────────────┐
│  📄 audio.mp3          │
│  ⏰ 01/15 14:30        │
│  转录内容预览...       │
└────────────────────────┘

悬停后：
┌────────────────────────┐
│  📄 audio.mp3    [×]   │  ← 红色删除按钮
│  ⏰ 01/15 14:30        │
│  转录内容预览...       │
└────────────────────────┘
```

### 注意事项
- ⚠️ **删除后无法恢复**，请谨慎操作
- ⚠️ 如果删除的是当前会话，页面内容会被清空
- ⚠️ 删除会同时移除所有相关数据（转录、总结、对话）

### 建议
- 定期清理不需要的记录
- 删除前确认不再需要
- 重要记录请提前备份

## 3️⃣ 对话历史永久保存

### 核心改进

**之前：**
```
1. 在音频A中对话："这段音频讲什么？"
2. 切换到音频B
3. 返回音频A
   ❌ 对话记录消失了！需要重新提问
```

**现在：**
```
1. 在音频A中对话："这段音频讲什么？"
   AI: "主要讨论了项目进度..."
   
2. 切换到音频B进行其他对话

3. 返回音频A
   ✅ 之前的对话完整显示！
   ✅ 可以继续追问："还有其他要点吗？"
```

### 自动保存时机
- **每次对话后** 自动保存
- **无需手动操作**
- **实时更新到文件**

### 何时恢复
- 切换历史记录时
- 服务器重启后
- 重新打开页面时

### 使用场景示例

**场景1：处理多个音频**
```
早上 9:00
- 上传会议录音1，进行5轮对话
- 上传会议录音2，进行3轮对话

下午 15:00
- 点击会议录音1
- ✅ 早上的5轮对话全部显示
- 可以继续深入提问
```

**场景2：服务器维护**
```
处理音频并对话
↓
关闭服务器维护
↓
重新启动服务器
↓
打开网页，点击历史记录
↓
✅ 对话记录完整恢复
```

**场景3：多天使用**
```
周一：上传音频A，对话10轮
周二：上传音频B，对话5轮

```

## 💡 使用技巧

### 技巧1：使用描述性文件名
```
好的文件名：
✅ 项目启动会议_2025-01-15.mp3
✅ 客户访谈_张三_01-16.mp3

不好的文件名：
❌ audio1.mp3
❌ 录音.mp3
❌ new_recording.mp3
```

### 技巧2：多轮深入对话
```
第一轮（概览）：
问："这段音频的主要内容是什么？"

第二轮（细节）：
问："关于XX部分，能详细说明吗？"

第三轮（分析）：
问："这些内容有什么潜在风险？"

✅ 所有对话都会保存
✅ 随时可以回顾完整对话历史
```

### 技巧3：定期整理
```
每周一次：
1. 查看历史记录列表
2. 删除不再需要的记录
3. 保持列表整洁
4. 提升加载速度
```

### 技巧4：善用会话切换
```
处理多个相关音频时：
1. 依次上传所有音频
2. 每个音频进行初步对话
```

## ❓ 常见问题

### Q1：删除记录后能恢复吗？
**A：不能。**删除是永久性的，请在删除前确认。如果有重要内容，建议先复制保存。

### Q2：对话历史能保存多久？
**A：永久保存。**除非手动删除历史记录，否则对话历史会一直保存在 `audio_history/history.json` 文件中。

### Q3：切换会话时对话历史加载慢？
**A：正常现象。**首次加载历史对话可能需要1-2秒，取决于对话数量。后续切换会更快。

### Q4：删除按钮看不到？
**A：需要悬停。**将鼠标移到历史记录项上，删除按钮会自动出现在右上角。

### Q5：点击"上传新音频"后能恢复吗？
**A：可以。**点击任意历史记录即可恢复之前的内容。"上传新音频"只是清空显示，不会删除数据。

### Q6：对话历史占用多少空间？
**A：很小。**每轮对话大约占用几百字节。即使进行数百轮对话，总大小也不超过几MB。

### Q7：可以导出对话历史吗？
**A：可以。**对话历史保存在 `audio_history/history.json` 文件中，可以手动复制该文件进行备份。

## 🎯 最佳实践

### ✅ 推荐做法

1. **有意义的命名**
   - 给音频文件取描述性名称
   - 包含日期、主题等信息

2. **定期清理**
   - 每周检查一次历史记录
   - 删除不再需要的记录
   - 保持列表整洁

3. **充分利用对话历史**
   - 不要担心切换会话
   - 对话历史会自动保存
   - 可以随时继续之前的讨论

4. **备份重要数据**
   - 定期备份 `audio_history` 文件夹
   - 重要对话可以截图或复制保存

### ❌ 避免做法

1. **频繁删除和重传**
   - 对话历史会永久保存
   - 不需要重新上传同一文件

2. **使用无意义文件名**
   - 历史记录中难以识别
   - 影响使用效率

3. **不备份就删除**
   - 删除后无法恢复
   - 重要数据应先备份
